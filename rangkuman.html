RANGKUMAN GIT git init: untuk inisialisasi git repository (membuat repository baru yang masih kosong). repository adalah hidden directory ketika git berjalan git status: melihat status project saat ini, melihat perubahan - perubahan yang terjadi pada respository ketika muncul info (file) is "untracked" -> artinya file tersebut adalah file baru git add ketika mau melakukan perubahan pada suatu file / project, maka project harus dimasukkan pada stage area (staging area) menggunakan command -> git add namafile ex: git add octocat.txt git add '*.jenisfile' : untuk menambahkan file-file dengan tipe yang sama git commit File yang dirubah masih berada pada staging area, masih belum tersimpan di respository. File dapat diubah (add, remove) di dalam stage area sebelum dimasukkan ke respository. Untuk menyimpan perubahan yang dilakukan di dalam stage, gunakan command -> git commit -m "isi pesan" -m : message git log: melihat history dari commit yang dilakukan (urut) git remote add remotename respositoryURL: untuk memasukkan (push) local repo ke GitHub server, diperlukan remote respository. ex: git remote add origin https://github.com/try-git/try_git.git memasukkan perubahan local ke origin repo (di GitHub) git push -u remotename default_local_branch_name: untuk memberi tahu git, tempat meletakkan commit yang dilakukan ex: git push -u origin master git pull origin master : untuk cek segala perubahan di GitHub respository dan pull down segala perubahan yang dilakukan git diff: melihat perbedaan yang terjadi git diff HEAD: untuk melihat perbedaan yang terjadi pada commit yang paling terakhir penggunaan diff lainnya adalah dapat melihat perubahan pada files yang sudah ada di staged staged files adalah file yang ready to be committed git diff --staged: melihat perubahan yang terjadi di staged git reset namafile: remove file tersebut dari stage (unstage) git checkout -- targetfile: untuk mengembalikan file ke kondisi saat commit terakhir dilakukan BRANCH ketika membuat program, biasanya developer akan membuat copy dari program yang dibuat (membuat branch). tiap branch dapat di commit secara terpisah. dan branch-branch dapat digabungkan kembali pada main master branch nya. commandnya adalah git branch namabranch git checkout namabranch_yang_mau_diswitch: untuk men-switch branch yang ada. maksud dari checkout, kita set posisi kita di branch yang diswitch tersebut. git rm: menghapus file yang diinginkan pada branch ex: git rm '*.txt' git rm -r 'namafile' -r: recursive git merge namabranch_yang_ingin_di_merge: menggabungkan branch ke main branch Sebelum melakukan merge branch, posisikan pointer di main branch dulu. dengan command-> git checkout namamasterbranch Setelah menggabungkan branch dengan main branch, branch dapat dihapus. dengan command git branch -d branchname: menghapus branch yang diingingkan (sesuai branchname) git push: memasukkan / push semua yang telah dikerjakan ke remote respository catatan tambahan: single quote dan double quotes sama.
